## 一、探索文档评估

### 1.1 文档优点

- **架构选型合理**：PyWebView + Vue 前后端分离方案，对 Python 开发者友好，匹配"快速构建小工具"定位
- **痛点抓得准**：准确识别了 tkinter/PySide 在拖拽、美化、打包上的历史问题
- **通信机制明确**：pywebview 的 js_api 桥接 + evaluate_js 双向通信方案简单直接
- **示例代码完整**：端到端可运行示例，降低了认知门槛

### 1.2 需修正/改进的问题

| 问题 | 原文档状态 | v2 改进方案 |
|:---|:---|:---|
| CDN 依赖 | Vue/Element Plus 通过 CDN 引入，离线不可用，无 Tree-shaking | 改用 Vite 构建，所有依赖本地化，产物仅包含实际使用的代码 |
| 前端工程化不足 | 单 HTML 文件，复杂工具难以维护 | Vite + Vue SFC 工程化，支持组件拆分、TypeScript、模块化 |
| 组件库选型 | Element Plus CDN，JS 运行时体积大，主题定制不够灵活 | 改用 DaisyUI + TailwindCSS，纯 CSS 无 JS 运行时开销，30+ 内置主题一键切换 |
| evaluate_js 脆弱 | Python 字符串拼接 JS 调用，易出错难调试 | 引入类型化事件总线，JSON 序列化 + 事件名约定，前后端双向类型约束 |
| 全局变量管理进程状态 | 多 global 变量 + 无线程锁 | 封装 ProcessManager，内置 threading.Lock 和状态机 |
| 无 uv 集成 | 使用 pip | 全流程 uv 管理依赖、虚拟环境、脚本运行 |
| 无 spec 文件管理 | 仅 CLI 命令 | 预置多场景 spec 模板（onedir/onefile/debug），自动检测前端产物路径 |
| 无项目脚手架 | 手动复制粘贴 | CLI 命令一键生成，自动初始化前后端项目 |
| 无标准化目录结构 | 2 文件示例 | 前后端分离的标准化工程结构 |
| 缺少日志系统 | print() | 基于 loguru 的结构化日志，控制台彩色输出 + 前端双通道，支持级别过滤、文件轮转、异常堆栈 |
| 缺少配置管理 | 窗口参数硬编码 | YAML 配置文件 + 环境变量覆盖 |
| 缺少错误处理规范 | try-except 过于简单 | 统一错误码体系 + Result 包装类型 + 全局异常拦截 |
| 无开发热更新 | 修改代码需重启 | 开发模式集成 Vite HMR，前端修改即时生效 |
| 无系统对话框 | 未涉及 | 封装 pywebview 原生文件/文件夹对话框 |
| 无单实例锁 | 可重复打开多个窗口 | 框架内置单实例检测，防止重复启动 |

## 二、框架定位与目标

### 2.1 定位

面向 Python 开发者的**桌面小工具快速开发框架**，核心价值：

1. **开发快**：脚手架一键生成前后端项目，专注业务逻辑，UI 用 DaisyUI 现成生态
2. **打包简单**：预置 PyInstaller spec 模板，uv 管理环境，一条命令出包
3. **体积可控**：DaisyUI 纯 CSS 无 JS 运行时，Vite Tree-shaking 优化，目标 15-30MB
4. **维护容易**：前后端分离，SFC 组件化，修改互不影响
5. **主题灵活**：DaisyUI 30+ 内置主题，一行配置切换明暗/多彩风格

### 2.2 非目标

- 不是大型桌面应用框架（不与 Electron/Tauri 竞争）
- 不支持移动端
- 不需要复杂的多窗口/多页面路由（单页应用即可）
- 不提供商业级热更新/自动升级机制

## 三、技术选型

| 层级 | 技术选择 | 说明 |
|:---|:---|:---|
| 桌面容器 | pywebview 5.0+ | 零额外环境依赖，Python 原生控制，原生拖拽支持 |
| 前端框架 | Vue 3 + Vite | SFC 组件化，HMR 热更新，Tree-shaking 产物优化 |
| CSS 框架 | TailwindCSS + DaisyUI | 纯 CSS 无 JS 运行时，30+ 内置主题，语义化组件类名 |
| 后端语言 | Python 3.10+ | 业务逻辑主力 |
| 包管理(Python) | uv | 项目初始化、依赖管理、脚本运行 |
| 包管理(前端) | bun | Vite 项目依赖管理 + JS 运行时，速度快、磁盘占用小 |
| 打包工具 | PyInstaller + spec 文件 | 标准化打包流程，模板化管理 |
| 配置格式 | YAML | 人类可读的配置文件 |

### 3.1 为什么选 DaisyUI + TailwindCSS 而非 Element Plus？

| 维度 | DaisyUI + TailwindCSS | Element Plus |
|:---|:---|:---|
| 运行时开销 | 纯 CSS，0 KB JS | JS 运行时 ~200KB+ |
| 主题切换 | 30+ 内置主题，一行 `data-theme="dark"` 切换 | 需自定义 CSS 变量 |
| 打包体积 | CSS 仅包含实际使用的类（PurgeCSS） | CSS + JS 全量引入 |
| 学习曲线 | Tailwind 原子化类名 + DaisyUI 语义化类名 | Element 专属 API |
| 定制能力 | Tailwind 无限定制 + DaisyUI 自定义主题 | 主题系统相对封闭 |
| 拖拽/交互 | 使用原生 HTML5 API 或轻量库如 SortableJS | 内置但体积大 |
| 离线可用 | Vite 构建后全部本地化 | CDN 模式需联网 |

### 3.2 为什么用 Vite 而非 CDN？

| 维度 | Vite 构建 | CDN 模式 |
|:---|:---|:---|
| 离线运行 | 产物自包含，100% 离线可用 | 依赖 CDN，首次需联网 |
| 开发体验 | HMR 热更新，修改即时生效 | 手动刷新，无模块化 |
| 产物体积 | Tree-shaking 裁剪未使用代码 | 全量加载 |
| 代码组织 | SFC 组件、模块化拆分 | 单 HTML 文件，复杂工具难维护 |
| 类型支持 | 支持 TypeScript | 不支持 |
| 前端依赖管理 | bun 版本锁定 | 无版本管理 |

**代价**：需要安装 bun（仅开发时，打包后不需要）。

#### F-003: 业务 API 基类 (app.py)

```python
from pywebvue import ApiBase, Result

class MyApi(ApiBase):
    def __init__(self):
        super().__init__()

    # 框架自动暴露给前端调用
    def health_check(self) -> Result:
        return Result.ok(data={"version": "1.0.0"})

    def process_file(self, file_path: str) -> Result:
        self.logger.info(f"Processing: {file_path}")
        if not os.path.exists(file_path):
            return Result.fail(ErrCode.FILE_NOT_FOUND, detail=file_path)
        # 业务逻辑...
        return Result.ok(data={"size": os.path.getsize(file_path)})

    # 拖拽回调（框架自动调用）
    def on_file_drop(self, file_paths: list[str]):
        for path in file_paths:
            result = self.process_file(path)
            if result.is_ok:
                self.emit("file_processed", result.data)
```

ApiBase 内置能力：
- `self.window` - pywebview 窗口引用
- `self.emit(event, data)` - 类型化事件推送（替代脆弱的 evaluate_js 拼接）
- `self.logger` - 结构化日志器（双通道：控制台 + 前端）
- `self.run_in_thread(func, *args)` - 后台线程便捷方法
- `self.config` - config.yaml 中的自定义业务配置
- `self.dialog` - 系统对话框（文件选择、文件夹选择）
- 所有公开方法自动暴露给前端，返回值必须是 `Result` 类型

#### F-004: 统一错误码体系

定义完整的错误码层级结构，Python 和前端共享同一套错误码定义。

**错误码格式**：`模块码(2位) + 错误码(3位)`

```python
# 框架内置错误码（Python 端定义，前端同步）
class ErrCode:
    """框架内置错误码"""
    # 00: 通用错误 (00xxx)
    OK = 0
    UNKNOWN = 1
    PARAM_INVALID = 2
    NOT_IMPLEMENTED = 3
    TIMEOUT = 4
    INTERNAL_ERROR = 5

    # 01: 文件系统 (01xxx)
    FILE_NOT_FOUND = 1001
    FILE_READ_ERROR = 1002
    FILE_WRITE_ERROR = 1003
    FILE_FORMAT_INVALID = 1004
    FILE_TOO_LARGE = 1005
    PATH_NOT_ACCESSIBLE = 1006

    # 02: 进程管理 (02xxx)
    PROCESS_START_FAILED = 2001
    PROCESS_ALREADY_RUNNING = 2002
    PROCESS_NOT_RUNNING = 2003
    PROCESS_TIMEOUT = 2004
    PROCESS_KILLED = 2005

    # 03: 网络/通信 (03xxx)
    API_CALL_FAILED = 3001
    API_NOT_READY = 3002

    @classmethod
    def to_msg(cls, code: int) -> str:
        """错误码转默认中文消息"""
        _MSG = {
            cls.OK: "success",
            cls.UNKNOWN: "unknown error",
            cls.PARAM_INVALID: "invalid parameter",
            cls.FILE_NOT_FOUND: "file not found",
            cls.FILE_READ_ERROR: "file read failed",
            cls.FILE_WRITE_ERROR: "file write failed",
            cls.FILE_FORMAT_INVALID: "unsupported file format",
            cls.FILE_TOO_LARGE: "file size exceeds limit",
            cls.PROCESS_START_FAILED: "failed to start process",
            cls.PROCESS_ALREADY_RUNNING: "process is already running",
            cls.PROCESS_NOT_RUNNING: "no running process",
            cls.PROCESS_TIMEOUT: "process execution timeout",
            cls.API_CALL_FAILED: "backend call failed",
            cls.API_NOT_READY: "backend is not ready",
        }
        return _MSG.get(code, "unknown error")

    @classmethod
    def is_user_error(cls, code: int) -> bool:
        """是否为用户操作引起的错误（可友好提示）"""
        return code in (cls.PARAM_INVALID, cls.FILE_NOT_FOUND,
                        cls.FILE_FORMAT_INVALID, cls.FILE_TOO_LARGE,
                        cls.PROCESS_ALREADY_RUNNING, cls.PROCESS_NOT_RUNNING)
```

**Result 包装类型**：

```python
from dataclasses import dataclass, field
from typing import Any

@dataclass
class Result:
    code: int
    msg: str = ""
    data: Any = None

    @classmethod
    def ok(cls, data: Any = None, msg: str = "success") -> "Result":
        return cls(code=ErrCode.OK, msg=msg, data=data)

    @classmethod
    def fail(cls, code: int, detail: str = "", msg: str = "") -> "Result":
        return cls(
            code=code,
            msg=msg or ErrCode.to_msg(code),
            data={"detail": detail} if detail else None,
        )

    @property
    def is_ok(self) -> bool:
        return self.code == ErrCode.OK
```

**前端类型同步**：

```typescript
// frontend/src/types/index.ts
// 与 Python ErrCode 保持同步
export const ErrCode = {
  OK: 0,
  UNKNOWN: 1,
  PARAM_INVALID: 2,
  FILE_NOT_FOUND: 1001,
  FILE_READ_ERROR: 1002,
  FILE_WRITE_ERROR: 1003,
  FILE_FORMAT_INVALID: 1004,
  FILE_TOO_LARGE: 1005,
  PROCESS_START_FAILED: 2001,
  PROCESS_ALREADY_RUNNING: 2002,
  PROCESS_NOT_RUNNING: 2003,
  PROCESS_TIMEOUT: 2004,
} as const;

export type ErrCodeType = typeof ErrCode[keyof typeof ErrCode];

export interface ApiResult<T = unknown> {
  code: number;
  msg: string;
  data: T | null;
}

// 前端调用辅助
export function isOk(result: ApiResult): result is ApiResult & { data: NonNullable<typeof result.data> } {
  return result.code === ErrCode.OK;
}
```

**用户自定义错误码**：

```python
# 用户在 app.py 中扩展错误码
class MyErrCode(ErrCode):
    # 10: 业务自定义 (10xxx)
    EXCEL_SHEET_NOT_FOUND = 10001
    DATA_VALIDATION_FAILED = 10002
    EXPORT_EMPTY_DATA = 10003

    @classmethod
    def to_msg(cls, code: int) -> str:
        msgs = {
```

#### F-005: 事件总线（前后端通信）

**Python 端（框架封装）**：

```python
# 所有 ApiBase 方法
self.emit("status_change", {"status": "running", "pid": 12345})
self.emit("log:add", {"level": "INFO", "message": "Processing row 42..."})
self.emit("progress:update", {"percent": 65})
self.emit("file:processed", {"path": "/tmp/data.xlsx", "rows": 1000})
```

事件命名约定：
- 使用 `模块:动作` 格式，如 `log:add`、`progress:update`、`status:change`
- 数据 payload 使用 snake_case JSON

**前端（框架内置）**：

```typescript
// frontend/src/event-bus.ts（框架自动生成，用户直接使用）
import { onMounted, onUnmounted } from "vue";

type EventCallback = (data: any) => void;

// 框架封装的事件监听 composable
export function useEvent(name: string, callback: EventCallback) {
  onMounted(() => {
    window.pywebvue.event.on(name, callback);
  });
  onUnmounted(() => {
    window.pywebvue.event.off(name, callback);
  });
}
```

```vue
<!-- 用户组件中使用 -->
<script setup lang="ts">
import { ref } from "vue";
import { useEvent } from "../event-bus";

const logs = ref<string[]>([]);
const status = ref("idle");

useEvent("log:add", (data) => {
  logs.value.push(`[${data.level}] ${data.message}`);
});

useEvent("status:change", (data) => {
  status.value = data.status;
});
</script>
```

**实现原理**：
Python 端 `self.emit()` 调用 `window.evaluate_js('__pywebvue_dispatch("event_name", {...})')`，
前端框架 JS 层 `__pywebvue_dispatch` 负责解析 JSON payload 并分发给已注册的监听器。

#### F-007: 子进程管理器 (ProcessManager)

封装 subprocess 管理，线程安全，支持多个独立进程实例：

```python
from pywebvue import ApiBase, ProcessManager

class MyApi(ApiBase):
    def __init__(self):
        super().__init__()
        self.pm = ProcessManager(self)  # 自动绑定 emit 推送

    def start_task(self, file_path: str) -> Result:
        if self.pm.is_running:
            return Result.fail(ErrCode.PROCESS_ALREADY_RUNNING)

        self.pm.start(
            cmd=["python", "task.py", "--input", file_path],
            cwd=".",                          # 工作目录
            on_output=self._on_output,        # 逐行回调
            on_complete=self._on_complete,    # 进程结束回调
        )
        return Result.ok()

    def _on_output(self, line: str):
        # ProcessManager 自动 emit("process:output", ...)
        # 用户可额外处理
        if "ERROR" in line:
            self.emit("log:add", {"level": "ERROR", "message": line})

    def _on_complete(self, returncode: int):
        if returncode == 0:
            self.emit("status:change", {"status": "done"})
        else:
            self.emit("status:change", {"status": "error"})
            self.emit("log:add", {"level": "ERROR", "message": f"Process exited with {returncode}"})

    def pause_task(self) -> Result:
        return self.pm.pause()

    def resume_task(self) -> Result:
        return self.pm.resume()

    def stop_task(self) -> Result:
        return self.pm.stop()
```

ProcessManager 核心能力：
- 后台守护线程运行子进程，不阻塞 GUI
- 内置 `threading.Lock`，线程安全的状态机
- 实时逐行读取 stdout/stderr，自动 emit 到前端
- 跨平台暂停/继续（Windows: `CREATE_NEW_PROCESS_GROUP` + `CTRL_BREAK_EVENT`，Unix: `SIGSTOP/SIGCONT`）
- 强制中止（`process.kill()`）
- 状态查询：`is_running`, `is_paused`, `pid`
- 可选：支持多实例（`ProcessManager.create("task_a")` / `ProcessManager.create("task_b")`）

#### F-016: 多进程实例管理

ProcessManager 支持命名多实例：

```python
class MyApi(ApiBase):
    def __init__(self):
        super().__init__()
        # 创建两个独立的进程管理器
        self.encode_pm = ProcessManager(self, name="encode")
        self.upload_pm = ProcessManager(self, name="upload")

    def start_encode(self):
        self.encode_pm.start(cmd=["python", "encode.py"])

    def start_upload(self):
        self.upload_pm.start(cmd=["python", "upload.py"])

    def stop_all(self):
        self.encode_pm.stop()
        self.upload_pm.stop()
```

```typescript
const api = window.pywebview?.api;

// 等待 pywebview 就绪
export async function waitForReady(timeout = 10000): Promise<void> {
  const start = Date.now();
  while (!window.pywebview?.api) {
    if (Date.now() - start > timeout) throw new Error("Backend not ready");
    await new Promise(r => setTimeout(r, 100));
  }
}

// 通用调用封装
export async function call<T = unknown>(

  ...args: unknown[]

}
```

```
    |
    v
[Vue 组件自动重渲染]
```

### 9.3 通信约束

- 前端调用 Python **必须** `await`，不允许同步调用
- Python 耗时操作 **必须** 在后台线程，不允许阻塞主线程（框架自动拦截）
- 数据传递仅支持 JSON 可序列化类型（str/int/float/bool/list/dict/None）
- 二进制数据（文件内容等）通过文件路径传递，不直接传输
- 事件 payload 建议不超过 1MB，大数据量使用分片或文件
